AI workflow pipeline

NutriPrompt does not send user input directly to an AI model. It enriches the request with rules, context and compatibility signals.

1
Structured form Goals, restrictions, preferences and real-life context.
2
RAG retrieval Relevant nutrition rules are retrieved before prompt generation.
3
Prompt builder Context, rules and profile tags are merged into a controlled prompt.
4
AI orchestration Gemini first, OpenAI fallback, structured mock mode for demos.
5
Output rendering JSON normalized into HTML, shopping logic and downloadable PDF.

Product capabilities

Each capability is implemented as a separate service layer to keep the project scalable, testable and easy to explain.

Service-oriented architecture

The backend is organized into focused service modules instead of placing all logic inside Django views.

ai/ Prompt builder, providers, JSON parser and validation.
rag/ Knowledge base, retrieval and context building.
vision/ OCR, label reading and compatibility analysis.
nutrition/ Fallback plans, meal rules and shopping logic.
profiles/ User profile classification and contextual tags.
presentation/ Output cleanup, HTML rendering and PDF presentation.
